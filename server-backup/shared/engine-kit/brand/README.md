# Brand Directory — Engine Kit

## Structure

Each brand has its own directory containing identity assets:

```
brand/
├── hiphop-world/
│   └── brand.json       — Brand identity manifest
├── pdx-business/
│   └── brand.json
└── pdx-gives/
    └── brand.json
```

## Brand JSON Format

Each `brand.json` contains:

```json
{
  "name": "HipHop.World",
  "domain": "hiphop.world",
  "site_key": "world",
  "site_type": "hiphop",
  "colors": {
    "primary": "#FFD700",
    "accent": "#E5B80B",
    "background": "#0B0B0B",
    "text": "#FFFFFF"
  },
  "fonts": {
    "brand": "Oswald",
    "heading": "Inter",
    "body": "Roboto"
  },
  "logo": {
    "svg": "logo.svg",
    "png_1024": "logo-1024.png"
  }
}
```

## Brand Pack Generation

Brand packs are generated by `BrandPackGenerator` (`---hiphop.world/includes/BrandPackGenerator.php`):

1. **SVG Analysis** — Extracts dominant colors from the uploaded logo using DOMDocument
2. **Palette Mapping** — Maps extracted colors to primary/accent/background roles
3. **Token Generation** — Creates `tokens.json` and `tokens.css` with `--brand-*` variables
4. **Favicon Generation** — Creates 16px and 32px favicons using Imagick or GD
5. **CSS Output** — Generates ready-to-use CSS variables

### CLI Usage
```bash
php cli/generate-brand-pack.php --svg=/path/to/logo.svg --site=world
php cli/generate-brand-pack.php --auto  # Auto-generate for all sites
```

## Brand Resolution

`brand-loader.php` (`---hiphop.world/includes/brand-loader.php`) resolves branding at runtime:

1. Check `engine_sites.branding` JSON for the site
2. Fall back to brand pack directory assets
3. Fall back to auto-generated branding from domain hash

The resolution chain ensures every site has branding, even if no brand pack has been explicitly created.

## Integration with Theme System

Brand packs feed into the engine-kit design system:
- Brand colors map to `--color-primary`, `--color-accent`, etc. in `theme-loader.css`
- Each brand's theme JSON (`design-system/themes/*.json`) documents the full token set
- The `ThemeLoader.set()` API activates themes at runtime
